    Learn¶

    How to install Zero?¶

    For installation instructions, see Installation.

    What is Reference?¶

    Reference (see the sidebar) is the main source of knowledge about the library. It contains everything you may need to understand how to use specific tools: API specification, tutorials and usage examples.

    How to import Zero?¶

    Simply do import zero and use what you need without accessing the submodules. For example, zero.evaluation is the correct way to access the function, while zero.learning.evaluation is not, because the structure of submodules is not stable yet.

    How to learn Zero?¶

    Zero, by design, is rather a “toolbox” than a “framework”, so you can learn things and start using them in almost any order. If you want to explore the library by yourself, this page is a good place to start.

    However, if you want to quickly learn “the most important” things, then understanding of this classification task is enough. To achieve this, you need to walk through some specific parts of Reference paying attention to explanations and examples, without diving into API details. Here are the things to learn:

    zero.learning.evaluation(*modules)

    Context-manager for models evaluation.

    zero.random.set_randomness(seed[, …])

    Set seeds and settings in random, numpy and torch.

    zero.random.get_random_state()

    Aggregate global random states from random, numpy and torch.

    zero.tools.ProgressTracker(patience[, min_delta])

    Tracks the best score, facilitates early stopping.

    zero.tools.Timer()

    Measures time.

    zero.hardware.get_gpus_info()

    Get information about GPU devices: driver version, memory, utilization etc.

    zero.data.concat(iterable)

    Concatenate items of the iterable along the first dimension.

    Congratulations! You are ready to apply Zero in practice. You can also visit this page and explore things that are not covered in the list above.
